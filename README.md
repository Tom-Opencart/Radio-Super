# Radio Super — Premium Music Player 🎵

Стильный, сверхкомпактный горизонтальный аудиопроигрыватель в стиле Glassmorphism с интерактивным 5-полосным спектроанализатором (имитация Music Wave) в реальном времени. 

Проект разработан как легкое статическое веб-приложение, готовое к размещению на любом веб-хостинге (включая GitHub Pages).

---

## ✨ Основные особенности

* **Ультракомпактный горизонтальный дизайн:** Высота плеера на десктопе составляет всего `155px`. Минималистичный интерфейс органично впишется на любую страницу.
* **Кастомная анимация Music Wave (Спектроанализатор):**
  * Визуализатор разделен на **5 независимых частотных блоков** (басы, нижняя середина, вокал/середина, верхняя середина, высокие частоты).
  * Толщина столбиков составляет ровно **1px** (для максимальной технологичности и воздушности).
  * Внутри каждого блока настроен уникальный профиль высот («небоскребы»), исключающий скучный овальный вид волны и создающий динамичную контрастную картинку.
  * Квадратичное сглаживание амплитуды убирает эффект «подвешенности», заставляя пики выразительно расти от самого основания.
  * В режиме паузы плеер переходит в режим ожидания с мягким «дыханием» силуэта города.
* **Кнопка управления в углу:** Кнопка Play/Pause вынесена в правый верхний угол вплотную к логотипу `music lab`, убирая лишнее нагромождение кнопок в центре.
* **Нижняя линия прогресса:** Линия проигрывания трека перенесена на самый низ панели управления и начинается ровно от правой границы обложки.
* **Плавная ротация обложек:** Обложки безрамочного формата сменяют друг друга по кругу с мягким кроссфейдом во время проигрывания. Интегрирован механизм обхода кэша браузера для мгновенного обновления изображений.
* **Адаптивность:** Полностью перестраиваемый интерфейс под мобильные экраны и планшеты.

## 🚀 Настройка и запуск

Плеер работает полностью на клиенте и не требует серверных скриптов (Node.js/PHP) для запуска в продакшене.

### 1. Добавление музыки
Поместите свои музыкальные файлы в папку `music/`. Рекомендуется использовать форматы `.mp3` или `.mp4` (AAC).

### 2. Добавление обложек
Поместите изображения обложек в папку `images/`. 

#### Именование файлов
Файлы должны называться по шаблону **`coverN`**, где `N` — порядковый номер начиная с `1`:
```
cover1.png
cover2.png
cover3.webp
cover4.jpg
…
```

#### Поддерживаемые расширения
| Расширение | Рекомендация |
|---|---|
| **`.webp`** | ✅ Рекомендуется — минимальный вес при высокой чёткости |
| **`.png`**  | ✅ Хорошо — без потерь качества, но тяжелее `.webp` |
| **`.jpg`**  | ✅ Допустимо — лёгкий, но с потерей качества |

#### Требования к изображениям
* **Соотношение сторон:** Строго **1:1** (квадратное изображение), чтобы избежать растягивания и искажений.
* **Рекомендуемый размер:** От **`300×300`** до **`500×500` пикселей**. Этого достаточно для высокой четкости на High-DPI (Retina) экранах, при этом сохраняется минимальный вес файла.
* **Без полей:** Обложки должны быть полноформатными, без белых рамок и полей по краям.

#### Регистрация в плейлисте
Каждую обложку нужно добавить в массив `"images"` файла `playlist.json`:
```json
"images": [
  "images/cover1.png",
  "images/cover2.png",
  "images/cover3.webp"
]
```
> ℹ️ Обложки ротируются автоматически с мягким кроссфейдом во время воспроизведения. Количество обложек не ограничено — добавляйте столько, сколько хотите.

### 3. Обновление плейлиста
Отредактируйте файл `playlist.json`, указав в нем ваши треки и картинки. Пример структуры:
```json
{
  "tracks": [
    {
      "file": "music/Your_Awesome_Song.mp3",
      "filename": "Your_Awesome_Song.mp3",
      "title": "Название трека",
      "artist": "Имя исполнителя",
      "duration": 241.0
    }
  ],
  "images": [
    "images/cover1.png",
    "images/cover2.png"
  ]
}
```

### 4. Размещение
Загрузите все файлы (кроме папки `.git`) на ваш хостинг. Главный файл для запуска — `index.html`.

> ⚠️ **Примечание для локальной разработки:** 
> Для работы визуализации звука (Web Audio API) браузеру требуется запуск страницы через локальный сервер (например, расширение Live Server в VS Code или запуск простой команды `python -m http.server 8000` в терминале). Прямой запуск двойным кликом по файлу (`file:///`) будет блокироваться политикой безопасности браузера (CORS).

## 🔒 Игнорирование медиафайлов (.gitignore)
Проект настроен так, чтобы тяжелые медиафайлы не перегружали репозиторий на GitHub. Правило в `.gitignore` автоматически исключает файлы музыкальных форматов из коммитов:
```gitignore
music/*
*.mp3
*.mp4
*.wav
```
Вы можете свободно наполнять папку `music/` локально — при обновлении репозитория эти файлы не будут отправлены на сервер.
